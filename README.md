# Сайт ресторана с интеграцией FUSIONPOS

Тиражируемый продукт: сайт ресторана с онлайн-меню, корзиной, самовывозом и доставкой,
админкой и интеграцией с POS-системой [FUSIONPOS](https://fusionpos.ru) (загрузка меню,
передача внешних заказов).

## Стек

- **Фронтенд** — Vue 3 + Vite + Pinia + vue-router (публичный сайт и админка в одном SPA)
- **Бэкенд** — Node.js + Express + Knex
- **БД** — MariaDB
- **Развёртывание** — Docker Compose (один инстанс = один ресторан)

## Возможности MVP

- Публичный сайт: шапка (логотип, баннер, телефон, часы работы), меню по категориям,
  карточки блюд с фото, корзина, оформление заказа (самовывоз / доставка),
  минимальная сумма и бесплатная доставка от суммы.
- Админка (`/admin`): настройки сайта и доставки, загрузка логотипа/баннера/фото блюд,
  синхронизация меню из FUSIONPOS, скрытие категорий и позиций, лента заказов со
  статусами и переотправкой заказа в POS.
- Интеграция с POS через слой драйверов: `mock` (демо-меню, для разработки и демонстрации)
  и `fusionpos` (реальный API, настраивается из админки).

## Быстрый старт (Docker)

```bash
cp .env.example .env
# задайте JWT_SECRET и пароль админа в .env
docker compose up -d --build
```

Сайт: http://localhost:8080 · Админка: http://localhost:8080/admin
(логин/пароль из `ADMIN_EMAIL` / `ADMIN_PASSWORD`).

После первого входа в админку откройте «Меню» и нажмите «Синхронизировать из FUSIONPOS» —
в демо-режиме загрузится тестовое меню.

## Локальная разработка

Требуется Node.js 20+ и запущенная MariaDB (`fp` / `fp_password`, БД `fp_site`).

```bash
# Бэкенд (порт 3000)
cd backend
cp .env.example .env
npm install
npm run migrate && npm run seed
npm run dev

# Фронтенд (порт 5173, проксирует /api и /uploads на бэкенд)
cd frontend
npm install
npm run dev
```

## Подключение реального FUSIONPOS

1. В Панели управления FUSIONPOS (Настройки → API Токены) создайте токен с правами:
   - **Меню** — «Просмотр» (для загрузки меню на сайт),
   - **Внешние заказы** — «Изменение» (для передачи заказов с сайта).
   Требуется подключённая опция «API (1С, сайты, боты)».
2. В админке сайта (Настройки → Интеграция FUSIONPOS) выберите драйвер «FUSIONPOS»,
   укажите базовый URL API и токен.
3. Пути эндпоинтов и маппинг полей заданы в `backend/src/pos/fusionpos.js`
   (константы `MENU_PATH`, `ORDERS_PATH`, функции `mapMenuResponse`, `buildOrderPayload`) —
   при получении официальной спецификации API их легко скорректировать в одном файле.

Заказы не теряются при недоступности POS: они сохраняются в БД со статусом «ошибка
отправки», из админки их можно отправить повторно.

## Тиражирование

Один ресторан = один инстанс `docker compose` со своим `.env` (порт, БД, секреты, токен
FUSIONPOS). Всё оформление (название, логотип, цвет, контакты, условия доставки)
настраивается через админку и хранится в БД, поэтому кодовая база общая для всех клиентов.

## Структура

```
backend/
  migrations/        # схема БД (knex)
  seeds/             # настройки по умолчанию, админ
  src/
    pos/             # драйверы POS: mock, fusionpos
    routes/          # public API, admin API
    services/        # синк меню, создание/отправка заказов
frontend/
  src/
    views/           # меню, оформление, заказ принят
    views/admin/     # логин, заказы, меню, настройки
    components/      # шапка, карточка блюда, корзина
    stores/          # pinia: корзина, настройки сайта
```
